Privacidad

Políticas de privacidad

Conoce cómo FastMarket utiliza y protege la información del cliente dentro del sistema frontend.

1. Datos que se solicitan

FastMarket puede solicitar nombre, correo, teléfono y dirección de entrega para simular el registro, inicio de sesión, compras y seguimiento de pedidos.

2. Almacenamiento actual

Los datos principales se guardan en la base de datos. El navegador puede conservar datos temporales como carrito de invitado. Esto permite simular el funcionamiento del sistema mientras se desarrolla el backend.

3. Uso de la información

Los datos se utilizan para mostrar el perfil del cliente, guardar direcciones, registrar pedidos y permitir el seguimiento de compras.

4. Seguridad

El almacenamiento actual es solo para pruebas frontend. En una versión real, las contraseñas no deben guardarse directamente en el navegador ni en texto plano. Con Spring Boot se debe implementar seguridad adecuada.

5. Eliminación de datos

El usuario puede cerrar sesión o borrar los datos desde el navegador. Más adelante, con backend, se podrá implementar eliminación de cuenta desde la base de datos.

6. Uso de mapas

La página de pedidos utiliza una integración visual de Google Maps para mostrar una ubicación aproximada basada en la dirección ingresada por el cliente.
